# 漫畫推薦系統 - 產品需求文件 (PRD)

## 1. 專案概述

- **背景與動機**
  許多漫畫愛好者常常面臨「書荒」的困境，不知道接下來該看哪一部作品。市面上的漫畫繁多，但往往缺乏針對個人口味的推薦或是方便的搜尋、分類與討論機制。本系統旨在打造一個專門推薦優質中文漫畫的平台，幫助讀者快速且精準地找到符合喜好的作品。
- **目標用戶**
  喜歡看漫畫的人。
- **核心價值主張**
  提供清晰的分類、強大的搜尋與相似推薦、社群書評機制以及熱門推薦榜，一站式解決漫畫讀者找書、評書的需求。

## 2. 功能需求

以下列出系統的核心主要功能與對應的使用者故事 (User Story)：

1. **推薦好看的中文漫畫**
   - 作為 **喜歡看漫畫的人**，我希望 **系統首頁能推薦我好看的中文漫畫**，以便 **我能在不知道看什麼的時候快速發掘新作品**。

2. **留言與書評機制**
   - 作為 **喜歡看漫畫的人**，我希望 **可以在每部漫畫的專屬頁面留下我的書評與心得**，以便 **與其他同好分享討論，並作為他人選書的參考**。

3. **種類選擇與過濾**
   - 作為 **喜歡看漫畫的人**，我希望 **系統提供詳細的種類選擇表（如熱血、戀愛、奇幻等）讓我進行篩選**，以便 **我能快速聚焦並挑選我當下想看的題材內容**。

4. **智慧搜尋與相似推薦**
   - 作為 **喜歡看漫畫的人**，我希望 **可以輸入關鍵字查詢想看的漫畫，且系統在展示搜尋結果時，也能一併提供類似內容或題材的漫畫選項**，以便 **我在尋找特定作品的同時，能順帶發掘更多可能喜歡的相似作品**。

5. **熱門推薦榜**
   - 作為 **喜歡看漫畫的人**，我希望 **能查看漫畫推薦排行榜**，以便 **我能了解目前社群中最熱門、評價最高或最具討論度的優質漫畫**。

6. **清晰的漫畫狀態與標籤標示**
   - 作為 **喜歡看漫畫的人**，我希望 **在漫畫列表與詳細資訊中，能清晰看到種類標示以及「已完結」或「連載中」的狀態**，以便 **我能根據自己的閱讀習慣（例如只看已完結的作品）做出判斷**。

## 3. 非功能需求

- **技術限制**
  - 後端與路由：使用 Python Flask 框架。
  - 前端渲染：使用 Jinja2 模板引擎進行頁面渲染，搭配基本 CSS/HTML/JS 實作互動。
  - 資料庫：使用 SQLite 作為關聯式資料庫，儲存漫畫資料、使用者、書評等資訊。
- **效能與安全考量**
  - **效能**：確保查詢漫畫清單、載入推薦榜單的回應時間在 2 秒內，提供流暢的瀏覽體驗。
  - **安全**：
    - 防範 SQL Injection：所有資料庫操作皆須使用參數化查詢或 ORM。
    - 防範 XSS：在渲染使用者提交的書評內容前，必須做好跳脫 (Escape) 處理。

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須有)**
  - 建立基本的漫畫資料庫架構（包含名稱、簡介、種類、完結/連載狀態）。
  - 首頁的靜態或簡單邏輯推薦區域。
  - 提供種類選擇表，可依據種類篩選漫畫。
  - 關鍵字搜尋功能。
  - 清晰展示漫畫資訊與狀態標籤。

- **Should Have (應該有)**
  - 使用者留言留書評的功能（可先不要求複雜會員系統，採簡單暱稱留言）。
  - 熱門推薦榜（根據評分或瀏覽次數排序）。
  - 在搜尋結果或漫畫介紹頁面，顯示「相似內容」漫畫。

- **Nice to Have (可以有)**
  - 基於使用者瀏覽/收藏歷史的個人化推薦演算法。
  - 完整的會員註冊/登入系統。
  - 進階的社群功能（書評按讚、回覆、討論串）。
